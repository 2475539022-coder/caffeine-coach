## 输出结构

定义该 Skill 输出给 Agent 或 UI 的结构。

```ts
type SkillOutput = {
  conclusion: string;
  reasons: string[];
  evidence: string[];
  suggestions: string[];
  confidence?: "low" | "medium" | "high";
  followUpQuestion?: string;
};
```

输出必须便于前台拆分展示：

- 结论；
- 原因；
- 依据；
- 建议行动；
- 兜底提示。

## 输出风格

- 面向普通用户；
- 少用技术词；
- 使用“可能”“倾向”“建议”“根据你的记录”；
- 不使用诊断式表达；
- 不夸大风险；
- 保持温和、克制、有陪伴感。

## 禁止事项

- 不做医疗判断；
- 不编造咖啡因含量；
- 不把搜索失败当作真实记录；
- 不绕过用户确认写入数据；
- 不直接修改设置；
- 不展示 Agent / Tool / Skill 等技术词给普通用户；
- 不输出无法由上下文或工具支持的结论。

## 失败兜底

当上下文不足或工具失败时：

- 明确说明信息不足；
- 给出低风险建议；
- 必要时请求用户补充记录、反馈或确认饮品；
- 不编造数据；
- 不阻塞主流程。

示例：

```text
目前记录还不够完整，只能做粗略估算。你可以先选择低因或半杯，并补充今天的饮品记录后再判断。
```
