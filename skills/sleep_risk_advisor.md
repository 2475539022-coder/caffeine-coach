# Sleep Risk Advisor Skill

用于回答“今晚还能不能喝咖啡 / 现在喝会不会影响睡眠 / 几点后不建议喝”这类睡眠风险与饮用建议问题。它服务 Caffeine Coach 核心链路中的“判断”和“建议”环节。

## 适用场景

- 用户问：“今晚还能喝咖啡吗？”
- 用户问：“现在喝会不会影响睡眠？”
- 用户问：“几点后不建议喝咖啡？”
- 用户问：“我睡前还能喝奶茶吗？”
- 用户问：“今天咖啡因还剩多少，会不会睡不着？”
- 用户问：“我现在适合再喝一杯吗？”
- 产品内部需要为首页今日饮用建议、喝前模拟或“为什么这样建议？”生成解释。

Router 应在问题核心是“睡眠风险、睡前残留、是否继续摄入、最晚饮用时间”时选择本 Skill。

## 不适用场景

- 用户只是想记录饮品，应交给 Drink Record Parser。
- 用户想查看一周总结，应交给 Weekly Review Writer。
- 用户想找替代饮品，应交给 Alternative Drink Recommender，或由本 Skill 给出简短替代后再转交。
- 用户询问医学诊断、治疗、疾病或药物处理咖啡因影响。
- 用户询问饮品库匹配、OCR、no_match 或自定义饮品沉淀。

## 输入上下文

必须使用 compact context，不读取完整 App 状态。

```ts
type IntakeRecord = {
  id: string;
  name: string;
  caffeineMg: number;
  time: string;
};

type SleepRiskAdvisorContext = {
  userProfile: {
    sleepTime: string;
    sensitivity: "sensitive" | "normal" | "tolerant";
    halfLife: number;
    reminderStrictness: string;
  };
  todayIntakeRecords: IntakeRecord[];
  remainingCaffeine: number;
  estimatedCaffeineAtSleep: number;
  sleepRisk: "low" | "medium" | "high";
  recentFeedbackSummary: {
    sleepAffectedDays: number;
    discomfortCount: number;
  };
};
```

字段说明：

- `sleepTime`：用户计划睡觉时间。
- `sensitivity`：当前敏感度解释后的简化状态。
- `halfLife`：当前半衰期估算小时数。
- `reminderStrictness`：提醒策略，影响文案保守程度。
- `todayIntakeRecords`：今日已记录摄入。
- `remainingCaffeine`：当前体内估算残留。
- `estimatedCaffeineAtSleep`：睡前预计残留。
- `sleepRisk`：睡眠风险等级。
- `recentFeedbackSummary`：近期睡眠受影响和即时不适反馈摘要。

## 必须使用的工具或数据

必须使用：

- 用户睡眠时间设置：用于判断距离睡觉时间和睡前残留意义。
- 今日摄入记录：用于说明今天已经喝了多少、是否已接近建议。
- 咖啡因残留计算：概念工具，对应当前项目已有 `totalRemaining`。
- 睡前预计残留：概念数据，对应当前项目已计算的 `sleepRemaining` / `estimatedCaffeineAtSleep`。
- 当前敏感度：来自设置、反馈和敏感度解释。
- 近期睡眠反馈：用于说明风险是否更保守。
- 近期即时不适反馈：用于说明是否建议更轻剂量。
- 睡眠风险判断规则：概念工具，对应当前项目已有 `riskLevel`。

如果由 Tool Use 层执行，可复用：

- `getTodayIntakeRecords`
- `getUserProfile`
- `calculateCurrentCaffeineStatus`
- `generateRecommendationSummary`
- `simulateDrinkBeforeAdding`，仅在“如果现在喝某饮品”时使用

## 可选工具或数据

- 最晚建议饮用时间：如果上下文已有，可用于回答“几点后不建议喝”。
- 低因饮品候选：如果需要替代方案，可从 Alternative Drink Recommender 或饮品库候选中获取。
- 敏感度解释摘要：用于补充“为什么对你更保守”。

## 输出结构

Skill 输出必须按以下结构组织。

```md
## 结论

建议 / 谨慎 / 不建议

## 原因

- 原因 1
- 原因 2
- 原因 3

## 建议

- 建议 1
- 建议 2

## 替代方案

- 替代 1
- 替代 2

## 风险提示

本建议仅用于生活管理和睡眠风险估算，不是医疗建议。
```

结论规则：

- `建议`：睡前预计残留低、风险低、近期反馈无明显不适。
- `谨慎`：风险中等、用户偏敏感、近期有睡眠受影响或即时不适反馈。
- `不建议`：睡前预计残留高、睡眠风险高、或用户高敏感且近期反馈不佳。

## 输出风格

- 结论先行。
- 表达具体，尽量带数值或时间。
- 使用“可能”“倾向”“建议”“根据你的记录”。
- 不吓唬用户。
- 不做绝对承诺。
- 不医学化。
- 不编造饮品咖啡因含量。
- 不承诺“一定不会失眠”。

示例语气：

```text
今晚建议谨慎。根据你今天的记录，当前仍有约 90mg 咖啡因残留，按 5 小时半衰期估算，睡前可能仍高于你的安心目标。建议今晚选择低因或无咖啡因饮品。
```

## 禁止事项

- 不得承诺一定不会失眠。
- 不得做医学诊断。
- 不得编造饮品咖啡因含量。
- 不得忽略用户敏感度。
- 不得忽略睡觉时间。
- 不得只给“少喝点”“看情况”这类泛泛建议。
- 不得建议用户通过药物处理咖啡因影响。
- 不得把没有记录解释为“一定没摄入”。
- 不得绕过用户确认写入记录或设置。
- 不得在普通用户输出中出现 Skill / Agent / Tool 等技术词。

## 失败兜底

当信息不足时：

- 明确缺少哪些信息；
- 给出保守建议；
- 引导用户补充睡眠时间或记录饮品；
- 不直接给确定结论；
- 不编造数据。

兜底模板：

```text
目前只能做粗略判断，因为缺少{缺失信息}。如果你今晚准备睡觉，建议先选择低因或无咖啡因饮品；补充睡眠时间和今天的饮品记录后，我可以再帮你估算睡前残留。
```

常见缺失：

- 缺少睡眠时间：无法估算睡前残留，应引导设置计划睡觉时间。
- 缺少今日记录：可说明“没有可用记录”，但不能断言用户没有摄入。
- 缺少敏感度：按普通敏感度粗略估算，并建议完善习惯设置。

## 验收用例

| 用例 | 用户问题 | 需要的上下文 | 预期输出要点 | 风险边界 |
|---|---|---|---|---|
| 1 | 今晚还能喝咖啡吗？ | 高敏感；睡前预计残留高；近期有睡眠受影响反馈 | 结论为“不建议”；说明睡前残留高、用户偏敏感、近期睡眠反馈影响判断；建议无咖啡因或低因 | 不说一定失眠，不做医学判断 |
| 2 | 现在还能喝吗？ | 下午；睡前预计残留低；正常敏感；近期无不适 | 结论为“建议”或“可以少量”；说明当前风险低；建议小杯或控制时间 | 不承诺一定不影响睡眠 |
| 3 | 几点后不建议喝咖啡？ | 缺少睡眠时间 | 说明缺少计划睡觉时间；给保守建议；引导设置睡眠时间 | 不给确定截止时间 |
| 4 | 我今天咖啡因还剩多少，会不会睡不着？ | 没有今日记录；有睡眠时间；普通敏感 | 说明没有可用摄入记录，无法确定真实残留；如 remainingCaffeine 为 0 可说明“记录内估算为 0” | 不断言用户实际没摄入 |
| 5 | 我睡前还能喝奶茶吗？ | 近期有 2 天睡眠受影响反馈；睡前预计残留中等 | 结论为“谨慎”；说明近期反馈会让建议更保守；建议半杯、低因或提前饮用 | 不编造奶茶咖啡因含量 |
| 6 | 你能保证我喝了也一定不会失眠吗？ | 任意上下文 | 明确不能保证；说明只能基于记录估算风险；建议谨慎 | 不承诺绝对结果 |

## 当前边界

- 当前不做真实 LLM 接入。
- 当前不修改咖啡因计算公式。
- 当前不修改饮品记忆、no_match、状态日历或反馈归属逻辑。
- 当前不生成复杂健康报告。
- 当前不做医疗建议。
- 当前替代饮品只给轻量建议，复杂替代推荐交给 Alternative Drink Recommender。

后续可优化：

- 与 Context Builder 连接，自动生成本 Skill 的 compact context。
- 与 Router 连接，处理相关自然语言问题。
- 将输出结构映射到首页建议卡、喝前模拟卡和解释折叠区。
