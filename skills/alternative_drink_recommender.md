# Alternative Drink Recommender Skill

用于在用户想喝点东西但当前不适合继续摄入高咖啡因时，给出低咖啡因、无咖啡因或非饮品替代方案。它服务 Caffeine Coach 核心链路中的“建议”和“替代行动”环节。

本 Skill 不做复杂推荐算法，不联网查询饮品，不编造咖啡因含量。

## 适用场景

- 用户问：“我现在还想喝点东西，有没有替代？”
- 用户问：“不喝咖啡可以喝什么？”
- 用户问：“晚上想喝奶茶，有没有低咖啡因选择？”
- 用户问：“有没有不影响睡眠的饮品？”
- 用户问：“我今天咖啡喝多了，还能喝什么？”
- 用户问：“想提神但不想喝咖啡怎么办？”
- 产品内部需要在睡眠风险中高、今日已接近推荐量或喝前模拟不建议完整摄入时给出替代方案。

Router 应在问题核心是“还想喝什么、替代咖啡、低因选择、无咖啡因选择、风险下的行动替代”时选择本 Skill。

## 不适用场景

- 用户只是记录一杯饮品，应交给 Drink Record Parser。
- 用户询问一周总结，应交给 Weekly Review Writer。
- 用户只问今晚还能不能喝，应优先交给 Sleep Risk Advisor。
- 用户询问医学治疗、药物或保健品建议。
- 用户要求保证不失眠。
- 用户要求联网查询具体品牌咖啡因含量。

## 输入上下文

必须使用 compact context，不读取完整 App 状态。

```ts
type AlternativeDrinkRecommenderContext = {
  userProfile: {
    sleepTime: string;
    sensitivity: string;
    halfLife: number;
    reminderStrictness: string;
  };
  currentState: {
    remainingCaffeine: number;
    sleepRisk: "low" | "medium" | "high";
    estimatedCaffeineAtSleep: number;
  };
  drinkOptions: {
    drinkLibrary: Drink[];
    frequentDrinks: FrequentDrinkMemory[];
    customDrinks: CustomDrink[];
  };
  recentFeedbackSummary: {
    sleepAffectedDays: number;
    discomfortCount: number;
  };
};
```

字段说明：

- `userProfile`：用户作息、敏感度和提醒策略。
- `currentState`：当前残留、睡眠风险和睡前预计残留。
- `drinkOptions`：可用于推荐的饮品库、常喝饮品和自定义饮品。
- `recentFeedbackSummary`：近期睡眠受影响和即时不适反馈。

## 必须使用的工具或数据

必须使用：

- 当前剩余咖啡因；
- 睡眠风险；
- 睡前预计残留；
- 用户敏感度；
- 近期睡眠反馈；
- 近期即时不适反馈；
- 饮品库中已有咖啡因含量；
- 常喝饮品和自定义饮品，仅在咖啡因含量可信时用于推荐。

如果由 Tool Use 层执行，可复用：

- `getUserProfile`
- `calculateCurrentCaffeineStatus`
- `searchDrinkLibrary`
- frequent drink memory 读取能力；
- custom drink 读取能力。

## 可选工具或数据

- Sleep Risk Advisor 的结论：用于判断是低风险、中风险还是高风险场景。
- 用户最近喝过但低咖啡因的饮品：用于更贴近日常习惯。
- no_match 记忆：只用于提示“未知”，不能作为低咖啡因依据。

## 推荐原则

1. 优先基于已有饮品库和用户常喝饮品。
2. 不编造饮品咖啡因含量。
3. 咖啡因含量未知时必须标注“未知”。
4. 当前风险高时，优先推荐无咖啡因或低咖啡因选择。
5. 当前风险中等时，可以推荐低咖啡因、小份量、提前饮用。
6. 当前风险低时，可以给出更宽松建议。
7. 推荐理由必须结合用户当前状态。
8. 推荐不能承诺“一定不影响睡眠”。
9. 无数据饮品不能被说成低咖啡因。
10. 非饮品替代应保持生活化，不写成医疗建议。

## 推荐类别

### 低咖啡因饮品

- 仅在饮品库或自定义饮品有明确 `caffeineMg` 时使用；
- 如果咖啡因未知，必须标注“咖啡因未知，需确认”；
- 可以建议小份量或半杯，但要说明仍需确认含量。

### 无咖啡因饮品

- 温水；
- 气泡水；
- 无咖啡因花草茶；
- 牛奶或热饮；
- 不虚构具体品牌含量。

### 非饮品替代

- 散步；
- 拉伸；
- 洗脸；
- 短暂休息；
- 调整环境光；
- 不写成医疗建议或治疗方案。

## 输出结构

Skill 输出必须按以下结构组织。

```md
## 结论

根据当前状态，建议选择低咖啡因 / 无咖啡因 / 暂缓摄入。

## 推荐选项

1. 选项名称
   - 咖啡因情况
   - 推荐理由
   - 注意事项

## 为什么这样推荐

结合当前剩余咖啡因、睡眠时间、敏感度、近期反馈说明。

## 不建议选项

列出当前不推荐的类型，例如高咖啡因咖啡、大杯奶茶、浓茶等。

## 风险提示

说明仅作生活管理建议，不保证一定不影响睡眠。
```

## 输出风格

- 先给可执行选择，再解释原因。
- 保持温和，不制造焦虑。
- 明确咖啡因含量未知时不能判断。
- 使用“建议”“可以优先考虑”“可能更稳妥”。
- 不承诺绝对安全。
- 不医学化。

示例语气：

```text
今晚更建议选无咖啡因饮品。你当前睡眠风险偏高，睡前预计仍可能有咖啡因残留，可以优先考虑温水、气泡水或无咖啡因花草茶。
```

## 禁止事项

- 不得编造咖啡因含量。
- 不得承诺一定不影响睡眠。
- 不得做医学建议。
- 不得推荐药物、保健品或治疗方案。
- 不得忽略用户当前风险。
- 不得把无数据饮品说成低咖啡因。
- 不得制造焦虑。
- 不得把 pending no_match 当作可信饮品选项。
- 不得绕过用户确认记录替代饮品。
- 不得在普通用户输出中出现 Skill / Agent / Tool 等技术词。

## 失败兜底

当信息不足时：

- 不知道睡眠时间：给更保守建议；
- 不知道饮品咖啡因：要求用户确认或选择已知饮品；
- 饮品库没有替代选项：推荐非饮品替代；
- 用户问“保证不失眠吗”：明确不能保证。

兜底模板：

```text
这杯饮品的咖啡因含量还不确定，我不能把它当作低咖啡因选项。你可以选择已知低因饮品，或先选无咖啡因饮品，比如温水、气泡水或无咖啡因花草茶。
```

## 验收用例

| 用例 | 用户问题 | 输入上下文 | 预期推荐 | 禁止输出 |
|---|---|---|---|---|
| 1 | 高风险用户晚上还想喝咖啡 | 睡眠风险高；睡前预计残留高；偏敏感 | 推荐无咖啡因饮品和非饮品替代；不建议完整咖啡 | 不说一定失眠，不推荐高咖啡因 |
| 2 | 中风险用户想喝奶茶 | 睡眠风险中；今日仍有残留 | 建议低因、小份量或暂缓；提醒确认咖啡因含量 | 不编造奶茶含量 |
| 3 | 低风险用户下午想提神 | 睡眠风险低；距离睡觉较远 | 可推荐小杯低因或已知低咖啡因选项，也可给非饮品提神 | 不承诺一定不影响睡眠 |
| 4 | 饮品库中没有明确低咖啡因含量 | 候选饮品 caffeineMg 未知 | 标注未知，建议用户确认或选择无咖啡因方案 | 不把未知说成低因 |
| 5 | 用户要求“保证不影响睡眠” | 任意上下文 | 明确不能保证，只能估算风险；给保守选择 | 不承诺保证 |
| 6 | 用户近期有心悸反馈 | discomfortCount 高；睡眠风险中或高 | 更保守，优先无咖啡因和非饮品替代 | 不忽略心悸反馈 |

## 当前边界

当前不做：

- 不接入外部饮品数据库；
- 不联网查询咖啡因；
- 不新增复杂推荐算法；
- 不做医疗建议；
- 不推荐药品或保健品；
- 不修改饮品记录逻辑；
- 不修改 no_match；
- 不修改常喝饮品；
- 不修改 UI。

当前依赖：

- 饮品库；
- 自定义饮品；
- 常喝饮品；
- 当前剩余咖啡因；
- 睡眠风险；
- 用户敏感度；
- 近期反馈。

后续可优化：

- 与 Sleep Risk Advisor 组合使用；
- 与 Context Builder 连接，生成替代推荐 compact context；
- 将推荐选项映射到喝前模拟或首页建议卡；
- 为低咖啡因选项建立可维护白名单。
