# 阶段 4.1：状态日历 + 反馈记忆 MVP 验收文档

## 1. 阶段名称

阶段 4.1：状态日历 + 反馈记忆 MVP

## 2. 阶段目标

本阶段目标是让 Caffeine Coach 能按天沉淀咖啡因摄入、睡前残留、风险等级和主观反馈，并通过状态日历进行轻量展示。

状态日历不是普通历史记录页，也不是复杂健康报告。它是记忆系统的前台展示入口，用于帮助用户回看“某一天发生了什么”，并为洞察页趋势复盘提供可追溯的日粒度依据。

## 3. 已完成能力

- 已支持 `dailyStatusMemory` 近 14 天派生状态重算。
- 已支持 `feedbackMemory` 主观反馈保存。
- 洞察页已加入状态日历入口。
- 状态日历已调整为简洁日历主视图。
- 点击日期后，会打开底部抽屉详情。
- 抽屉详情展示当天摄入、睡前预计残留、风险等级、记录明细和反馈摘要。
- 支持从某一天详情中补录反馈。
- 睡眠类反馈归属前一天。
- 即时不适反馈归属当天。

## 4. 验收通过项

| 验收项 | 结果 | 说明 |
| --- | --- | --- |
| 新增记录后状态日历更新 | 通过 | 新增记录会触发当天状态重算，并影响近 14 天状态快照。 |
| 删除记录后状态日历重算 | 通过 | 删除记录后会重新计算今日摄入、睡前残留和状态日历数据。 |
| 设置修改后近 14 天状态重算 | 通过 | 修改睡眠时间、敏感度或半衰期相关设置后，近 14 天状态会基于原始数据重新计算。 |
| 补录反馈后对应日期显示已反馈 | 通过 | 保存反馈后，对应日期在状态日历中出现反馈标记。 |
| 抽屉详情与原始记录一致 | 通过 | 抽屉中的记录明细从原始摄入记录按日期过滤，不依赖派生快照伪造。 |
| 未影响首页、记录页、喝前模拟、我的页 | 通过 | 本阶段改动聚焦记忆与状态日历，不改变其他页面主链路。 |
| 未实现阶段 4.2 内容 | 通过 | 未新增 noMatchMemory、drinkUsageMemory、常喝饮品排序或补库入口。 |
| `npm run build` | 通过 | 构建通过。 |

## 5. 修复项

本次验收中修复了反馈归属问题。

修复前，反馈保存没有明确区分“昨晚睡眠反馈”和“今日即时不适反馈”的归属日期。

修复后：

- 睡眠类反馈写入前一天。
- 心悸、焦虑、手抖、胃不舒服、提神评分写入反馈当天。
- 不改变 `dailyStatusMemory` / `feedbackMemory` 数据结构。

这符合记忆系统设计中的边界规则：睡眠反馈默认归属到上一睡眠周期或前一摄入日，即时不适反馈归属到反馈发生当天。

## 6. 当前边界

阶段 4.1 暂不做以下内容：

- `noMatchMemory`
- `drinkUsageMemory`
- 常喝饮品排序
- 补库入口
- 复杂敏感度自动校准
- 真实 LLM
- 云端同步

这些能力保留到后续阶段，不进入当前 MVP 验收范围。

## 7. 阶段结论

阶段 4.1 验收通过。

Caffeine Coach 现在已经具备按天沉淀摄入、睡前残留、风险等级和主观反馈的基础记忆能力，并通过状态日历完成了前台展示与反馈补录闭环。
