# Stage 5.1：Sleep Risk Advisor Skill

## 1. 阶段名称

Stage 5.1：Sleep Risk Advisor Skill

## 2. 阶段目标

本阶段目标是把“今晚还能不能喝咖啡 / 现在喝会不会影响睡眠 / 几点后不建议喝”这类任务沉淀为一个可复用 Skill。

它不是新增业务功能，也不改变现有咖啡因计算、日历写入、反馈归属或饮品记忆逻辑，而是为后续 Router / Agent 调用建立稳定的任务说明书。

Sleep Risk Advisor 服务 Caffeine Coach 核心链路中的两个环节：

- 判断：根据今日摄入、睡前残留、睡眠时间和敏感度判断风险；
- 建议：给出是否继续摄入、是否需要减量、是否选择低因或无咖啡因饮品的行动建议。

## 3. Skill 适用场景

本 Skill 适用于用户或产品内部需要回答以下问题：

- “今晚还能喝咖啡吗？”
- “现在喝会不会影响睡眠？”
- “几点后不建议喝咖啡？”
- “我睡前还能喝奶茶吗？”
- “今天咖啡因还剩多少，会不会睡不着？”
- “我现在适合再喝一杯吗？”
- 首页今日饮用建议；
- 喝前模拟结果解释；
- “为什么这样建议？”的原因与依据生成。

不适用于：

- 只是记录饮品；
- 一周总结或趋势复盘；
- 复杂替代饮品推荐；
- 饮品库匹配、OCR、no_match 或自定义饮品沉淀；
- 医学诊断、治疗、疾病或药物相关问题。

## 4. 输入上下文

Sleep Risk Advisor 必须使用 compact context，不读取完整 App 状态。

```ts
type IntakeRecord = {
  id: string;
  name: string;
  caffeineMg: number;
  time: string;
};

type SleepRiskAdvisorContext = {
  userProfile: {
    sleepTime: string;
    sensitivity: "sensitive" | "normal" | "tolerant";
    halfLife: number;
    reminderStrictness: string;
  };
  todayIntakeRecords: IntakeRecord[];
  remainingCaffeine: number;
  estimatedCaffeineAtSleep: number;
  sleepRisk: "low" | "medium" | "high";
  recentFeedbackSummary: {
    sleepAffectedDays: number;
    discomfortCount: number;
  };
};
```

字段含义：

- `sleepTime`：用户计划睡觉时间；
- `sensitivity`：当前敏感度简化状态；
- `halfLife`：当前半衰期估算；
- `reminderStrictness`：提醒策略；
- `todayIntakeRecords`：今日摄入记录；
- `remainingCaffeine`：当前体内估算残留；
- `estimatedCaffeineAtSleep`：睡前预计残留；
- `sleepRisk`：睡眠风险等级；
- `recentFeedbackSummary`：近期睡眠受影响和即时不适反馈摘要。

## 5. 工具 / 数据依赖

必须使用：

- 用户睡眠时间设置；
- 今日摄入记录；
- 咖啡因残留计算；
- 睡前预计残留；
- 当前敏感度；
- 近期睡眠反馈；
- 近期即时不适反馈；
- 睡眠风险判断规则。

如果由现有 Tool Use 层执行，可以复用：

- `getTodayIntakeRecords`
- `getUserProfile`
- `calculateCurrentCaffeineStatus`
- `generateRecommendationSummary`
- `simulateDrinkBeforeAdding`，仅用于“如果现在喝某饮品”场景

本阶段只文档化 Skill，不新增复杂工程集成。

## 6. 输出格式

Skill 输出必须遵循以下结构：

```md
## 结论

建议 / 谨慎 / 不建议

## 原因

- 结合当前剩余咖啡因、预计睡前残留、睡眠时间、用户敏感度和近期反馈说明原因。

## 建议

- 给出可执行建议，例如减少摄入量、提前饮用、选择低因或今晚停止高咖啡因饮品。

## 替代方案

- 给出低咖啡因或无咖啡因替代方案。

## 风险提示

本建议仅用于生活管理和睡眠风险估算，不是医疗建议。
```

结论判断口径：

- `建议`：睡前预计残留低、风险低、近期反馈无明显不适；
- `谨慎`：风险中等、用户偏敏感、近期有睡眠受影响或即时不适反馈；
- `不建议`：睡前预计残留高、睡眠风险高，或用户高敏感且近期反馈不佳。

## 7. 输出风格

- 结论先行；
- 表达具体，尽量带数值或时间；
- 使用“可能”“倾向”“建议”“根据你的记录”；
- 不吓唬用户；
- 不做绝对承诺；
- 不医学化；
- 不编造饮品咖啡因含量；
- 不承诺“一定不会失眠”。

## 8. 禁止事项

本 Skill 必须遵守：

- 不得承诺一定不会失眠；
- 不得做医学诊断；
- 不得编造饮品咖啡因含量；
- 不得忽略用户敏感度；
- 不得忽略睡觉时间；
- 不得只给“少喝点”“看情况”这类泛泛建议；
- 不得建议用户通过药物处理咖啡因影响；
- 不得把没有记录解释为“一定没摄入”；
- 不得绕过用户确认写入记录或设置；
- 不得在普通用户输出中出现 Skill / Agent / Tool 等技术词。

## 9. 失败兜底

当信息不足时，Skill 需要：

- 明确缺少哪些信息；
- 给出保守建议；
- 引导用户补充睡眠时间或记录饮品；
- 不直接给确定结论；
- 不编造数据。

兜底表达示例：

```text
目前只能做粗略判断，因为缺少计划睡觉时间。今晚如果接近休息时间，建议先选择低因或无咖啡因饮品；补充睡眠时间后，我可以再帮你估算睡前残留。
```

常见兜底场景：

- 缺少睡眠时间：无法估算睡前残留，不给确定截止时间；
- 缺少今日记录：可以说明“记录内估算不足”，但不能断言用户实际没有摄入；
- 缺少敏感度：按普通敏感度粗略估算，并引导完善习惯设置；
- 工具失败：说明暂时只能给保守建议，不暴露底层错误。

## 10. 验收用例

| 用例 | 用户问题 | 需要的上下文 | 预期输出要点 | 风险边界 |
|---|---|---|---|---|
| 1 | 今晚还能喝咖啡吗？ | 高敏感；睡前预计残留高；近期有睡眠受影响反馈 | 结论为“不建议”；说明睡前残留高、用户偏敏感、近期睡眠反馈影响判断；建议无咖啡因或低因 | 不说一定失眠，不做医学判断 |
| 2 | 现在还能喝吗？ | 下午；睡前预计残留低；正常敏感；近期无不适 | 结论为“建议”或“可以少量”；说明当前风险低；建议小杯或控制时间 | 不承诺一定不影响睡眠 |
| 3 | 几点后不建议喝咖啡？ | 缺少睡眠时间 | 说明缺少计划睡觉时间；给保守建议；引导设置睡眠时间 | 不给确定截止时间 |
| 4 | 我今天咖啡因还剩多少，会不会睡不着？ | 没有今日记录；有睡眠时间；普通敏感 | 说明没有可用摄入记录，无法确定真实残留；如残留估算为 0，仅说明“记录内估算为 0” | 不断言用户实际没摄入 |
| 5 | 我睡前还能喝奶茶吗？ | 近期有 2 天睡眠受影响反馈；睡前预计残留中等 | 结论为“谨慎”；说明近期反馈会让建议更保守；建议半杯、低因或提前饮用 | 不编造奶茶咖啡因含量 |
| 6 | 你能保证我喝了也一定不会失眠吗？ | 任意上下文 | 明确不能保证；说明只能基于记录估算风险；建议谨慎 | 不承诺绝对结果 |
