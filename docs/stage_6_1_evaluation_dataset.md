# Stage 6.1 Evaluation Dataset

## 1. 阶段名称

Stage 6.1：测试集与标准答案

## 2. 阶段目标

本阶段基于 Stage 6.0 的 EvaluationTask 协议，构建一套覆盖正常输入、模糊输入、信息不足、安全越界和多意图场景的评测集。

这套测试集用于后续模块级评测、规则与工具测试、端到端任务验收，以及 Bad Case 归因和复测。

## 3. 新增数据集

本阶段新增 5 个评测数据集：

| 数据集 | 类别 | 条数 | 重点覆盖 |
| --- | --- | ---: | --- |
| `evaluation/datasets/drink_recognition_cases.json` | 饮品识别 | 16 | 记录意图、饮品匹配、模糊容量、no_match、模拟与真实记录区分 |
| `evaluation/datasets/sleep_risk_cases.json` | 睡眠风险 | 13 | 睡前残留、敏感度、睡眠时间缺失、不确定剂量、安全承诺边界 |
| `evaluation/datasets/personalization_cases.json` | 个性化 | 12 | 周复盘、敏感度解释、反馈归属、常喝饮品、数据不足 |
| `evaluation/datasets/safety_cases.json` | 安全边界 | 12 | 心脏不适、怀孕、过敏、药物、催吐、严重不适 |
| `evaluation/datasets/multi_intent_cases.json` | 多意图 / 端到端 | 7 | 任务拆分、写入确认、周复盘 + 替代建议、安全优先 |

总计：60 条评测任务。

## 4. 覆盖范围

### 4.1 饮品识别

覆盖：

- 标准记录输入；
- 品牌和饮品库匹配；
- 模糊容量；
- 相对时间；
- 自定义饮品；
- 搜索无结果；
- no_match 必须用户确认；
- “想喝”和“已经喝”的区分；
- 喝前模拟不能当成真实摄入。

### 4.2 睡眠风险

覆盖：

- 明确剂量和睡眠时间；
- 今日记录读取；
- 高敏感用户；
- 慢代谢用户；
- 没有今日记录；
- 缺少睡眠时间；
- 不知道咖啡因含量；
- 不承诺一定不会失眠；
- 工具异常时的降级处理。

### 4.3 个性化

覆盖：

- 近 7 天摄入复盘；
- 睡眠反馈；
- 即时不适反馈；
- 敏感度解释；
- 提神效果变化；
- 常喝饮品；
- 高风险时间；
- 数据不足时的有限结论；
- 不能把相关性说成确定因果。

### 4.4 安全边界

覆盖：

- 明显身体不适；
- 超高摄入；
- 怀孕；
- 疑似过敏；
- 药物或治疗请求；
- 保证不失眠；
- 严重心悸、胸痛、呼吸困难；
- 催吐等高风险行为。

安全类任务要求系统保持生活方式建议边界，不做医学诊断，不推荐药物或治疗方案。

### 4.5 多意图

覆盖：

- 记录后判断睡眠风险；
- 周复盘后推荐替代饮品；
- 未知饮品创建确认；
- 摄入较多且有心慌反馈；
- 补记历史饮品后复盘；
- 想喝但未喝的意图区分；
- 喝前模拟后推荐替代。

多意图任务的重点不是生成长回答，而是明确拆分任务、确认写入、处理安全优先级。

## 5. 标准答案结构

每条任务均遵循 `EvaluationTask` 协议，并包含以下标准答案字段：

- `expectedIntent`: 预期意图；
- `expectedSkill`: 预期 Skill 或 Skill 组合；
- `expectedTools`: 预期工具或规则能力；
- `requiredContextFields`: 必须读取的上下文字段；
- `expectedSteps`: 预期处理步骤；
- `expectedOutput.requiredFields`: 输出中必须包含的字段；
- `expectedOutput.expectedDecision`: 预期决策；
- `expectedOutput.expectedFacts`: 预期事实；
- `expectedOutput.prohibitedClaims`: 禁止出现的表述；
- `safetyRequirements`: 安全要求；
- `recoveryExpectation`: 错误恢复触发条件与动作；
- `passCriteria`: 通过标准。

标准答案不要求逐字一致，评审重点是意图、步骤、事实、安全和任务完成情况。

## 6. 标注规范

本阶段新增 `evaluation/annotation_guide.md`，用于说明人工评审口径。

标注规范覆盖：

- 路由正确；
- 字段解析正确；
- 工具调用正确；
- 上下文完整；
- 安全通过；
- 个性化通过；
- 可解释通过；
- 任务完成；
- 模糊案例处理；
- 评审意见不一致时的处理方式。

## 7. 当前边界

本阶段只构建评测数据和标注规范，不做以下事项：

- 不执行模型评测；
- 不修改 Router；
- 不修改 Skill；
- 不修复测试暴露的问题；
- 不修改业务逻辑；
- 不接入真实 LLM；
- 不新增业务页面。
